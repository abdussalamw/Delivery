هذا الدليل يشرح كيفية تحديث النظام ورفعه إلى السيرفر (VPS) لضمان بقاء الواجهة والميزات محدثة.

## 🛠️ خطوات التحديث (Deployment Workflow)

عند إجراء أي تعديلات برمجية على جهازك المحلي (Local XAMPP)، اتبع الخطوات التالية لرفعها للموقع:

### 1. الرفع إلى GitHub
قم بتنفيذ الأوامر التالية من المجلد الرئيسي للمشروع على جهازك:
```powershell
# إضافة التعديلات
git add .

# حفظ التعديلات برسالة وصفية
git commit -m "توضيح التعديلات التي قمت بها"

# الرفع للمستودع
git push origin main
```

### 2. سحب التحديثات على السيرفر (VPS)
اتصل بالسيرفر عبر SSH ثم قم بتحديث الكود:
```bash
# الدخول إلى مجلد المشروع
cd /root/delivery

# سحب التحديثات من GitHub
git pull origin main
```

### 3. إعادة تشغيل الحاويات (عند الضرورة)
إذا قمت بتعديل ملفات الإعدادات أو ملفات Docker، قد تحتاج لإعادة التشغيل:
```bash
# إعادة تشغيل الخدمات
docker-compose restart
```
*ملاحظة: ملفات PHP يتم تحديثها فورياً بمجرد عمل `git pull` لأنها مرتبطة كـ Volumes.*

## 🔒 ملاحظات أمنية هامة
- **ملفات الجلسة:** تم استثناء مجلد `auth_info/` من الرفع لضمان عدم تسريب بيانات اتصال الواتساب الخاصة بك.
- **إعدادات الدومين:** الدومين مرتبط عبر Nginx Reverse Proxy (HestiaCP) إلى منفذ Docker الداخلي.
